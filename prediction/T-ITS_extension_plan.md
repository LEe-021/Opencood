## 二、方法改进（共4项）

### 2.1 改进1：IMM-UKF预测阶段协方差传播

#### 当前问题

PR1_predict_IMM_UKF.py第160-177行中，多步预测仅传播均值 `imm.x`，完全丢弃协方差 `imm.P`：

```python
current_state = imm.x.copy()      # ✅ 取了均值
# ❌ 但没有取 imm.P，也没有在预测中传播协方差

for i in range(1, steps + 1):
    pred_cv = cv_transition(current_state, t_pred)     # 只推了均值
    mixed_pred_state = current_mu[0] * pred_cv + ...    # 只混合了均值
```

均值相同但协方差不同的两个预测，补偿置信度应完全不同：
- 预测A: x=10.0 ± 0.1m（低延迟，滤波充分）→ 应高度信任
- 预测B: x=10.0 ± 0.8m（高延迟，滤波不充分）→ 应降低信任
- 当前代码对两者产生完全相同的补偿结果

#### 改进内容

**(a) 滤波结束时获取完整混合协方差**

IMM框架的混合协方差应包含模型间散布项（当前代码缺失）：

```
P_mixed = Σ μ_j · P_j + Σ μ_j · (x_j − x_mixed)(x_j − x_mixed)ᵀ
```

- 第一项：各子滤波器协方差的加权和
- 第二项：模型间预测不一致带来的额外不确定性
- 缺少第二项会导致协方差被系统性低估

**(b) 多步预测中通过UT传播协方差**

对每个预测步，用Unscented Transform将协方差通过非线性状态转移函数传播：

```python
for step in range(1, PRED_STEPS + 1):
    for j, fx_j in enumerate([cv_transition, ctrv_transition, ctra_transition]):
        # 均值传播
        x_pred_j = fx_j(current_state, t_pred)
        # 协方差传播（通过sigma点）
        P_pred_j = propagate_covariance_via_ut(x_j, P_j, t_pred, fx_j)

    # 按演化后的模型权重混合
    mixed_state = Σ mu_j · x_pred_j
    mixed_cov   = Σ mu_j · P_pred_j + Σ mu_j · (x_pred_j - mixed_state)(x_pred_j - mixed_state)ᵀ
```

#### 涉及代码修改

- PR1_predict_IMM_UKF.py：predict()方法重写，增加协方差传播逻辑
- 输出JSON格式扩展：每个预测步增加"covariance"字段（6×6矩阵）

#### 对论文的贡献

为不确定性感知融合提供数学基础，使预测结果从"确定性点估计"升级为"概率分布估计"。

### 2.2 改进2：预测阶段模型权重马尔可夫演化

#### 当前问题

PR1_predict_IMM_UKF.py第173-177行，用滤波结束时的固定权重 `current_mu` 用于所有预测步：

```python
current_mu = imm.mu.copy()  # 滤波结束时的权重

# step 1 (50ms) 和 step 6 (300ms) 使用完全相同的权重
mixed_pred_state = current_mu[0] * pred_cv + current_mu[1] * pred_ctrv + current_mu[2] * pred_ctra
```

#### 改进内容

通过马尔可夫转移矩阵前向传播模型概率：

```python
mu_pred = current_mu.copy()
for step in range(1, PRED_STEPS + 1):
    mu_pred = M.T @ mu_pred            # 马尔可夫前向演化
    mu_pred = mu_pred / mu_pred.sum()  # 归一化

    # 用演化后的权重混合各模型预测
    mixed_state = Σ mu_pred[j] · pred_states[j]
```

#### 物理意义

预测时间越长，运动模式切换的可能性越大，模型权重趋向均匀分布（不确定性增大）：

```
滤波结束时:    [0.70, 0.20, 0.10]  → 明确倾向CV（直行）
预测50ms后:    [0.65, 0.23, 0.12]
预测150ms后:   [0.53, 0.28, 0.19]
预测300ms后:   [0.38, 0.33, 0.29]  → 趋向均匀，不确定性增大
```

#### 涉及代码修改

- PR1_predict_IMM_UKF.py：多步预测循环中增加马尔可夫权重更新

#### 对论文的贡献

使长时域预测的不确定性建模更加合理，尤其在高延迟场景下改善预测精度和协方差估计的可靠性。

### 2.3 改进3：基于不确定性的软加权融合

#### 设计理念

不是决定"用不用"RSU数据（硬门控），而是告诉检测网络"这些数据有多可靠"（软加权）。

**为什么不用硬门控**：RSU点云的核心价值在于补充ego车辆盲区。即使补偿有误差，保留有误差的点云仍比完全丢弃更有利于检测。硬门控可能在高延迟场景下过度丢弃点云，导致目标漏检。

#### 三层设计

**第一层 — 逐点补偿置信度计算**

对目标包围盒内每个补偿后的点，基于预测协方差计算置信度分数：

```python
def compute_point_confidence(point_offset, P_comp):
    """
    point_offset: 点到目标中心的偏移 (dx, dy)
    P_comp: 预测协方差矩阵 (6×6)
    """
    dx, dy = point_offset
    sigma_pos = sqrt(P_comp[0,0] + P_comp[1,1])
    sigma_yaw = sqrt(P_comp[3,3])

    # 杠杆臂效应：距中心越远，角度误差放大越大
    effective_sigma = sqrt(sigma_pos**2 + (dx**2 + dy**2) * sigma_yaw**2)

    # sigmoid映射，温和且不会产生极端值
    w = 1.0 / (1.0 + (effective_sigma / sigma_ref) ** 2)
    return w   # 输出范围 (0, 1]
```

不同条件下的自然行为：

| 条件 | 中心点 w | 边缘点 w | 效果 |
|---|---|---|---|
| 50ms + 直行 | ~0.98 | ~0.95 | 近乎无损融合 |
| 150ms + 弯道 | ~0.80 | ~0.55 | 完整保留目标轮廓，边缘适度降权 |
| 300ms + 急转弯 | ~0.60 | ~0.25 | 仍保留目标不丢失，网络知道边缘不可靠 |

关键特性：即使在最高不确定性条件下，w值仍大于0（不为零），保证点云不丢失。

**第二层 — 特征维度扩展**

将置信度作为附加特征通道输入PointPillars。修改 pillar_vfe.py 的特征构造过程：

```
当前特征: [x, y, z, intensity]              → 4维
扩展特征: [x, y, z, intensity, confidence]   → 5维

其中:
  - ego车辆的点: confidence = 1.0（无补偿损失）
  - RSU补偿后的点: confidence = w_i（基于协方差计算）
```

涉及修改的文件：
- pillar_vfe.py：将 `num_input_features` 从4改为5
- basedataset.py：在数据加载阶段为每个点拼接confidence特征

**第三层 — 检测网络自适应学习**

PointPillars内部的PointNet编码器在训练过程中自动学习：当confidence低时，降低对该点几何细节的依赖，但仍利用其存在性信息。无需修改网络结构，仅通过数据驱动实现自适应加权。

#### 涉及代码修改

- PR1_predict_IMM_UKF.py：输出协方差矩阵
- PR2_compensate_all_steps.py：计算逐点置信度并附加到点云数据
- pillar_vfe.py：扩展输入特征维度
- basedataset.py：数据加载时拼接confidence通道

#### 对论文的贡献

将卡尔曼滤波理论中的协方差信息与感知融合桥接，从"确定性补偿"升级为"概率性补偿"。这构成一个独立的理论贡献点，且实现复杂度适中。

### 2.4 改进4：马尔可夫转移矩阵运动状态自适应

#### 当前问题

PR1_predict_IMM_UKF.py第123-127行，转移矩阵对所有车辆和时刻完全相同：

```python
M = np.array([
    [0.90, 0.08, 0.02],
    [0.05, 0.90, 0.05],
    [0.02, 0.08, 0.90]
])
```

#### 改进内容

根据当前运动状态动态调整对角线概率：

```python
def adaptive_markov(base_M, current_state):
    M = base_M.copy()
    v, yaw_rate, a = current_state[2], current_state[4], current_state[5]

    if abs(yaw_rate) < 0.02 and abs(a) < 0.5:
        # 高速直行 → 增强CV自转移概率
        M[0,0] = 0.95; M[0,1] = 0.04; M[0,2] = 0.01
    elif abs(yaw_rate) > 0.1 and abs(a) > 0.5:
        # 弯道加减速 → 增强CTRA自转移概率
        M[2,2] = 0.95; M[2,0] = 0.02; M[2,1] = 0.03
    elif abs(yaw_rate) > 0.1:
        # 弯道匀速 → 增强CTRV自转移概率
        M[1,1] = 0.95; M[1,0] = 0.03; M[1,2] = 0.02

    M = M / M.sum(axis=1, keepdims=True)
    return M
```

#### 涉及代码修改

- PR1_predict_IMM_UKF.py：在滤波循环中根据状态动态调整M矩阵

#### 对论文的贡献

使模型切换更加灵敏，减少模式识别的响应延迟。可作为消融实验的对比项。

## 七、方法改进间的逻辑关系

```
改进1: 协方差传播          ──→  改进3: 软加权融合
       (预测不确定性量化)          (不确定性指导融合)
              ↑                          ↑
改进2: 权重演化              │     需要可靠的协方差
       (更准确的模型概率)    │     作为输入
              ↑              │
改进4: 自适应马尔可夫  ──────┘
       (更灵敏的模式切换)
```

四个改进形成一条完整链路：
1. **改进1**提供协方差 → 使预测从确定性变为概率性
2. **改进2**使权重随预测时间演化 → 使长时域预测更合理
3. **改进4**使马尔可夫矩阵适配运动状态 → 使模型切换更灵敏
4. **改进3**利用1-3的输出指导融合 → 实现不确定性感知的最终目标
