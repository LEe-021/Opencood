## 三、必须解决的审稿问题及修改计划

以下按优先级逐项说明修改方案，兼顾审稿意见和目标期刊的要求。

### 3.1 P0-1：补充对比实验（最关键）

#### 审稿人要求

> "The authors should include at least two to three competitive baselines." — R2
> "compared against stronger asynchronous cooperative perception baselines" — R3

#### 修改方案

建立**三级对比体系**，覆盖早期/中间/晚期融合三个层级：

| 编号 | 对比方法 | 融合层级 | 补偿原理 | 实现难度 |
|---|---|---|---|---|
| B1 | 无补偿直接融合 | 早期融合 | 无 | 已有 |
| B2 | CV点云补偿 | 早期融合 | 恒速线性外推+刚体变换 | 已有（PR1_predict_CV_KF） |
| B3 | 边界框级补偿 | 晚期融合 | 边界框线性/CTRV外推 | **需新增** |
| B4 | CoBEVFlow或SyncNet | 中间融合 | BEV特征流/P-LSTM | **需新增** |
| B5 | 本文方法（IMM-UKF点云补偿） | 早期融合 | 多模型状态外推+刚体变换 | 已有 |

**最小可接受组合**：B1 + B2 + B3 + B5（不含学习型方法）
**稳妥组合**：B1 + B2 + B3 + B4 + B5（全部）

#### 对比实验的技术说明

**(a) B3 边界框级补偿（Object-level）**

这是最容易实现且最有说服力的对比。思路：对RSU检测到的3D边界框参数（中心、尺寸、航向）进行时间外推，而不是对点云进行变换。外推后的边界框与ego检测结果进行融合（NMS/匹配）。

```python
# 伪代码：边界框级补偿
def bbox_level_compensation(bbox_rsu, delta_t, motion_model='cv'):
    # 从RSU检测框提取状态
    x, y, z, w, l, h, yaw = bbox_rsu

    # 根据运动模型外推
    if motion_model == 'cv':
        x_new = x + vx * delta_t
        y_new = y + vy * delta_t
    elif motion_model == 'ctrv':
        # CTRV外推
        ...

    return [x_new, y_new, z, w, l, h, yaw_new]
```

优势：可以直接展示"点级补偿"相对"框级补偿"的增益，证明保留原始几何信息的价值。

**(b) B4 特征级补偿（Feature-level）**

如果完整实现CoBEVFlow/SyncNet有困难，建议：
- 复现**简化的BEV特征补偿**：对RSU的BEV特征图进行基于光流的时序对齐
- 或直接引用公开代码（CoBEVFlow有GitHub仓库），在OpenCOOD框架中集成

在论文中需要说明：由于本文聚焦早期融合，中间融合方法的对比主要用于展示**不同融合层级**在时延条件下的性能差异，而非证明某个层级绝对优于另一个。

#### 预期对比结果表

| 方法 | 融合层级 | AP@0.5 (0ms) | AP@0.5 (150ms) | AP@0.5 (300ms) | AP下降 |
|---|---|---|---|---|---|
| B1 无补偿 | 早期 | 0.92 | 0.71 | 0.57 | 38% |
| B2 CV点云补偿 | 早期 | 0.92 | ~0.85 | ~0.78 | ~15% |
| B3 边界框补偿 | 晚期 | ~0.88 | ~0.82 | ~0.75 | ~13% |
| B4 特征流补偿 | 中间 | ~0.90 | ~0.84 | ~0.77 | ~13% |
| B5 本文方法 | 早期 | 0.92 | ~0.92 | ~0.91 | <2% |

> 注：B3/B4/B5的数值为预期估计，实际以实验结果为准。

### 3.4 P1-1：时延建模改进

#### 审稿人要求

> "deltaT is treated as a known deterministic scalar... Consider using a more sophisticated latency model" — R2

#### 修改方案

**(a) 在方法中将ΔT从确定量扩展为测量量**

在方法章节中增加讨论：

> "在实际车路协同系统中，延迟ΔT可通过时间戳差值计算获得。ego车辆记录RSU数据的接收时刻t_recv，与RSU数据包中携带的采样时刻t_stamp相减，即可获得包含通信延迟和采样相位差的广义延迟。该延迟值受网络波动影响存在随机性，本文在实验中通过固定延迟和随机延迟两组测试评估方法的鲁棒性。"

**(b) 增加随机延迟实验**

| 延迟模式 | 设置 | 目的 |
|---|---|---|
| 固定延迟 | 50/100/150/200/250/300 ms | 评估方法在不同延迟量级下的性能 |
| 均匀随机延迟 | U(100, 300) ms | 模拟网络波动 |
| 正态随机延迟 | N(150, 50) ms，截断至[50, 300] | 模拟典型通信延迟分布 |
| 突发延迟 | 基础100ms，10%概率突发至500ms | 模拟网络拥塞 |

> 注意：根据之前的讨论，暂不实现自适应时延估计，但在实验中通过上述随机延迟测试来评估方法对时延不确定性的鲁棒性。

### 3.5 P1-2：z轴不变假设的处理

#### 审稿人要求

> "urban roads are not perfectly flat. Gradients, speed bumps, road camber and uneven surfaces all produce measurable z-displacement and pitch variation" — R2

#### 修改方案

不在方法中实现完整SE(3)补偿，而是：

**(a) 在方法章节中明确假设的适用范围**

> "本文方法假设目标车辆运动主要发生在地面平面内，即高度方向（z轴）及俯仰角、横滚角保持不变。该假设适用于相对平坦的城市交叉口和普通国省干线公路路面。对于存在显著纵坡的路段，可在后续工作中扩展为SE(3)变换。"

**(b) 在实验中增加坡度敏感性测试**

在现有仿真中人为注入z轴扰动，测试不同坡度（0%、3%、6%）下性能变化：

```python
# 在PR2_compensate_all_steps.py中注入z轴扰动
z_noise = road_grade * distance_traveled  # 根据行驶距离和坡度计算z偏移
pitch_noise = arctan(road_grade)           # 俯仰角变化
```

如果结果显示3%坡度下性能下降<5%，则可作为"假设合理"的支撑证据。

### 3.6 P1-3：上游检测/跟踪敏感性分析

#### 审稿人要求

> "the sensitivity of the method to upstream tracking or detection errors is not analyzed" — R3

#### 修改方案

新增"上游感知质量影响分析"实验：

| 上游输入类型 | 设置 | 目的 |
|---|---|---|
| 真值框（Oracle） | 使用ground truth bbox | 上界性能 |
| 注入噪声框 | 位置噪声σ=0.1/0.3/0.5m，航向噪声σ=1/3/5° | 评估检测框噪声影响 |
| 漏检模拟 | 随机丢弃5%/10%/20%目标 | 评估漏检影响 |
| ID切换模拟 | 随机5%/10%帧的跟踪ID互换 | 评估跟踪稳定性影响 |

#### 关联到之前讨论的改进

这个分析恰好可以与之前讨论的"基于不确定性的软加权融合"结合——当上游检测质量差导致预测协方差增大时，软加权机制会自动降低补偿点云的置信度，形成自然的质量保护。

## 四、方法改进建议（结合之前讨论）

### 4.1 推荐实施的方法改进

| 改进项 | 对应审稿意见 | 难度 | 建议 |

### 5.2 写作风格注意事项

《中国公路学报》的写作规范与IEEE期刊有显著差异：

| 维度 | IEEE风格 | 中国公路学报风格 |
|---|---|---|
| 语言 | 英文 | **中文**（专业术语首次出现时附英文原文） |
| 引言 | 直接切入技术问题 | **需从国家战略/行业需求切入**（交通强国、智慧公路） |
| 公式 | 密集、编号完整 | **适度**，重点公式编号，推导过程可适当简化 |
| 图表 | 英文标注 | **中文标注**，图题/表题用中文 |
| 参考文献 | 以英文文献为主 | **中英文混合**，需适当引用中文交通工程文献 |
| 贡献表述 | 技术贡献为主 | **兼顾理论意义和工程应用价值** |

### 5.3 引言建议结构

```
第1段：国家战略背景
  交通强国战略 → 智慧公路是重要方向 → 公路交叉口感知是关键环节

第2段：技术挑战
  车路协同感知可扩展感知范围 → 但通信时延导致时空错位
  → 早期融合保留最完整信息但对时延最敏感

第3段：现有方法不足
  现有时延补偿方法主要针对晚期/中间融合 → 点级补偿研究不足
  → 直接拼接异步点云产生ghosting伪影 → 检测性能退化

第4段：本文工作
  提出面向车路协同早期融合的点级运动补偿框架
  → IMM-UKF多模型状态外推 + 刚体变换 + 空间对齐融合

第5段：贡献列表
  (1) 提出实例级点云时延补偿框架
  (2) 建立完整补偿管线并分析各环节误差影响
  (3) 多场景多方法系统对比验证
```
